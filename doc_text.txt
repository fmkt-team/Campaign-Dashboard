01
프로젝트 배경 및 목표
Background & Goals
1.1  왜 만드는가
현재 마케팅팀은 캠페인 성과 데이터가 GA4, 유튜브 스튜디오, 광고 플랫폼, 엑셀에 분산되어 있어
리더 보고 시 매번 수동으로 취합 → 시간 낭비 + 맥락 손실 발생.
또한 타팀/리더에게 캠페인 현황을 공유할 때 일관된 포맷이 없어 커뮤니케이션 비효율 존재.
1.2  무엇을 만드는가
멀티 캠페인을 하나의 플랫폼에서 운영·모니터링하는 관리자 대시보드
타팀·리더가 공유 링크로 접근하는 읽기 전용 뷰어 페이지
캠페인별 성과 데이터를 구조화하여 관리하는 Convex 백엔드

convex/
  ├── schema.ts                  # 전체 테이블 스키마 정의
  ├── campaigns.ts               # query: list, getById / mutation: create, update, remove
  ├── kpis.ts                    # query: getByWeek / mutation: upsert

  ├── traffic.ts                 # query: getWeekly, getDaily / mutation: upsert

5.3  데이터 입력 방식 (MVP 기준)
Phase 1: Admin UI에서 직접 수동 입력 (Convex mutation 호출)
Phase 2: Google Sheets → Convex 동기화 (HTTP Action + Sheets API Webhook)
Phase 3: GA4 API 직접 연결 (Convex Scheduled Action)
💡 Convex 장점: 실시간 useQuery 훅으로 데이터 변경 시 뷰어 페이지 자동 반영 (polling 불필요).
Admin이 KPI를 저장하면 공유 링크를 열고 있는 뷰어 화면에 즉시 업데이트됨.

07
UI/UX 디자인 원칙
Design Principles
7.1  디자인 컨셉: Glassmorphism + Monochrome
■  Glassmorphism: 반투명 유리 질감 카드 + 블러 배경 레이어로 깊이감 표현.
   배경은 어두운 그라디언트 (흑/다크그레이 계열), 카드는 white/8~12% 투명도 + backdrop-blur-md.
   카드 테두리: white/15~20% 얇은 1px 라인으로 유리 엣지 표현.
■  Monochrome: 컬러를 배제하고 흰색·회색·검정 명도 차이만으로 위계 구성.
   강조 포인트는 white (100%) — 서브는 white/60% — 보조 텍스트는 white/40%.
   차트도 단일 컬러(흰색 + 투명도 변주)로 통일. 상태값(증가/감소)만 예외적으로 색상 허용.
■  레퍼런스 무드: Apple Vision Pro UI / Linear.app 대시보드 / Vercel Analytics
